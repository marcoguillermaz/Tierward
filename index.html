TierwardA governed process for building with Claude Code

Claude generates. You or your team decides. A governed, phased process for building with Claude Code, tier by tier.

Quick Start
View on GitHub
🔒

Mechanical enforcement

Claude can't commit until you approve the plan. STOP gates and a commit gate enforce that mechanically. A Stop hook also blocks task completion until your tests pass. These are hard blocks in the control flow, not configuration you tune.

📋

26 audit skills

Executable multi-step programs that run inside Claude Code. Security, performance, architecture, accessibility, compliance. Each produces a structured report.

🪜

Tier by tier

Start at Tier 0 with a Stop hook and nothing else. Upgrade one tier at a time as your team's needs grow. Never pay the overhead of a workflow you don't need.

🔌

MCP-native

Ships an MCP server alongside the CLI. Any MCP-aware client (Claude Desktop, Cursor, VS Code) can read your project's governance state without running the CLI.
